GODS / PIETY SYSTEM (classes/gods.py, editor/gods_tab.py)
- WorldData singleton: 8 gods in 4 opposed pairs, global action counters
- World balance per dichotomy (action counts)
- Piety drift from witnessed opposing acts, scaled by nearby opposing piety
- Lose god when piety hits 0; aligned actions reinforce (+0.005)
- Action dispatch records all successful actions in god counters
- Piety influences StatWeightedBehavior (aligned actions boosted, opposed penalized)
- Piety feeds reward function (gaining piety = positive, losing = negative)
- proselytize() action: CHR contest to convert creatures with no/low deity
- World state flags for quest/dialogue conditions
- Gods editor tab with aligned/opposed action lists
- NOT YET: cutscene/proselytize animations

SPATIAL MEMORY & GOALS (classes/creature/_goals.py)
- GoalMixin: spatial memory dict (known_locations) + goal state per creature
- remember_location(): stores purpose -> [(map, x, y, tick)] with dedup and cap (10/purpose)
- update_spatial_memory(): auto-learns tile purpose on 500ms tick
- set_goal/clear_goal: hierarchical RL goal tracking (purpose, target, zone_id, tick)
- goal_distance/goal_progress/at_goal: Manhattan distance tracking and progress delta
- pick_goal_target: selects best known location for a purpose (current map > other map)
- direction_to_goal: unit vector toward goal target, cross-map aware via map_graph
- plan_cross_map_route(): returns (exit_x, exit_y) of linking tile toward target map
- Cross-map distance: exit_tile_dist + transitions * 50 (when map_graph provided)
- PurposeZoneRegistry (classes/purpose_zones.py): fast zone lookup, nearest zone, auto-load from DB
- GoalNet (classes/goal_net.py): 75→256→128→17 purposes, selects goals via RL policy
- TorchGoalNet: training version with PPO, wired into MAPPO + PPO training loops
- Goal observation in action model (self_goal section: 21 floats)
- Goal-conditioned reward: progress (+0.3/tile closer), completion (+3.0)
- Purpose proximity reward: reward^(1+1/d) near purpose tiles, -0.25 away
- update_spatial_memory scans all visible tiles (sight range), not just current tile
- Goal + action weights saved together in nn_models (goal_weights BLOB column)
- NOT YET: goal model resume with dimension remapping (currently loads strict)
