FUTURE / NOT STARTED
=====================
- Presentation layer: stylized text, SFX/music, cutscene engine
- 3D rendering mode (Doom-style BSP from tile data)
- Per-creature learnable reward weights (personality via 0.5-2.0 bounded scales)
- Shop/vendor NPCs (distinct from peer-to-peer trade)
- Pathfinding for NPC cross-map travel (map_graph exists, behavior module doesn't)
- Mourning / grief system: when a high-sentiment relationship dies (direct
  witness or via rumor), survivor gets a `grief` stat mod scaled by
  sentiment * ln(count+1) for a bereavement window (3-7 game days).
  Effects: reduced productivity, lower food intake, lower CHR on social,
  reduced combat aggression. Reward signal m_grief/grief_penalty teaches
  the NN to protect allies. Death tile becomes a place of interest
  (funeral-like emergent behavior). Ties into existing ghost system
  (_ghost_loved_uid visits during grief window). Training via dedicated
  "loved ones vs monster" scenario: spawn 2-8 creatures with pre-seeded
  strong positive relationships against monsters; survivor grief reward
  teaches protective behavior. See memory/project_mourning_system.md.

COMPLETED SYSTEMS
=================

Core Architecture
- Trackable base class: all game state objects extend it for automatic save/load
- RelationshipGraph: centralized sentiment + rumor + deceit graph (Trackable singleton)
- Pickle save/load via Trackable.all_instances() — no special cases
- WorldObject._by_map spatial index with WeakSet
- Map spatial grid for O(cell) creature queries
- Dynamic zoom (0.25x-4.0x), 3/4 perspective rendering
- Event-driven vs polling: creature social topology cached via GRAPH
  generation counter, spatial memory movement-triggered, water tick
  conditional, regen ticks unregister when full
- Observation space expanded with 33 pre-allocated monster slots
  (5 slots x 6 fields + 3 summary floats). Stable OBSERVATION_SIZE
  across creature-only and monster-enabled curriculum phases.

Creature System
- Single Creature class (no subclasses): players, NPCs, monsters all Creature
- Stats: base/derived/mods/active layers, D&D-style modifiers, opposing contests
- 32-action space: MOVE (auto-direction) + 31 utility/combat/social/economy actions
- Auto walk/run/sneak: walk default, run when hostile in sight, sneak as persistent toggle
- Movement: 8-dir walk/run/sneak, flee, follow, map transitions (enter/exit)
- Combat: melee, ranged (with ammo), grapple, cast_spell (8 spells in DB), block stance
- Weapons: infinite_ammo flag for natural (monster) ranged weapons
- Social: intimidate, deceive (context-gated + rumor poisoning + deceit tracking), trade (auto + proposed), bribe, steal (auto-resolve + familiarity modifier), share_rumor, talk, call_backup, proselytize
- Deceit system: outstanding lies tracked in RelationshipGraph, revelation via social interactions, deception_stress penalty signal, long-term honesty incentive
- Economy: harvest (5 resource types), farm, dig, process (9 recipes), craft (ItemFrames), job (7 job types), trade with loan fallback, REPAY_LOAN
- Inventory: pickup (auto + gold + coalesce), smart drop (lowest value/weight), auto-equip (NN only), use_item (consumables + food + Meat with spoilage/cannibalism), transfer_item (adjacent inventories)
- Reproduction: propose_pairing, force_pairing, egg system, genetics (14-gene chromosomes, Mendelian inheritance), fecundity curves
- Encumbrance: S-curve penalty above 100% carry weight (regen slowed), movement blocked at 200% with struggle sound feedback
- Swimming: can_swim from species DB or learned (innate 20%, witness 10%/tick, survival 2%/tick)
- Occupation system: SLEEP (6-8hr, gradual stamina/mana restore, wake on noise/damage/expiry, 1% sleepwalk), JOB (1hr, purpose effects + wage each tick, hostile interrupts)
- Durability: weapons and armor lose 1 durability per hit, break at 0 (removed from equipment)
- Dialogue: full conversation tree system with conditions, effects, quest triggers (loaded in training via DB)
- Traps: creator-immune (trap_creator_uid stored on tile, checked before trigger)
- Covered tiles: block cross-cover vision
- Meat drop on death: every dying LivingThing drops a Meat item with species tag and 48hr spoil window
- meat_value species attribute: hunger restoration when consumed (auto-derived from size if NULL)
- Cannibalism detection: eating meat where meat.species == self.species broadcasts -15 rumor to every witness within sight range, -0.1 piety if devoted; _cannibalism_events counter for reward

Monster System
- Monster class: top-level (NOT Creature subclass), shares CombatMixin/
  MovementMixin/InventoryMixin/RegenMixin, stubs for social methods
  so shared combat paths work against monster targets
- MonsterSpecies DB: 9 seeded archetypes with stats + lifecycle/
  territory/dominance/diet/sprite configuration
- Pack class (Trackable): separate M/F dominance hierarchies, auto-
  rank by STR+VIT+AGL, event-driven signals (sleep/alert/cohesion/
  role/target_position) with delta-threshold broadcast
- Territory model: territory_scales toggle, effective formula
  (scales with pack.size, modulated by cohesion), sampled target
  positions from N(center, effective_std), territories_overlap
  via 3-sigma circle intersection
- Pack-vs-Pack rules: neutral by default, hostile only on territory
  overlap, merge condition (same species + combined <= split_size/2)
- Collapse on alpha death: species flag that dissolves the pack into
  solitary 1-member packs when alpha dies (bees/ants)
- MonsterNet: 73 -> 256 -> 128 -> 64 -> 11 actions, INT-gated mask
  (instinct/feral/aware bands), diet-gated HARVEST
- PackNet: 14 -> 64 -> 32 -> 6 (sleep/alert/cohesion sigmoids + 3-way
  role softmax)
- Monster observation builder: 73 floats across 9 sections
- Creature-side monster perception: 5 monster slots + 3-float summary
  populate when monsters visible
- Heuristic monster + pack policies (priority cascade)
- Imitation dataset generator (monster_imitation.py)
- Imitation pretraining (editor/simulation/monster_pretrain.py):
  TorchMonsterNet + TorchPackNet, supervised cross-entropy training,
  DAgger iteration, .npz export to models/
- MonsterTrainer (editor/simulation/monster_train.py):
  TorchMonsterPolicy with actor-critic heads, per-monster rollout
  buffers, PPO update with masked logits, attach_to_sim replaces the
  heuristic with NN-driven actions, build_from_pretrained loads .npz
  weights into the training policy, export_weights writes
  inference-format .npz back to models/
- PackNet online REINFORCE: MonsterTrainer also collects per-pack
  trajectories (aggregated member rewards), runs REINFORCE with
  critic baseline, entropy regularization, gradient clipping
- Location rumor system (territory + purpose): RelationshipGraph
  .add_location_rumor / location_rumors_of / get_location_rumors /
  best_location_rumor. Creatures record territory rumors on sight
  of monsters (-15 sentiment on pack center) and purpose rumors on
  standing on purpose tiles. share_location_rumor propagates with
  0.6 confidence decay per hop.
- Live viewer: renders monster hexagons (red-tinted, alpha gets gold
  ring), pack territory circles (dim red outline), monster/pack
  count in info panel
- LeaguePool (editor/simulation/league_pool.py): rolling snapshot
  manifest with max_snapshots FIFO eviction, component-scoped
  sampling, latest/clear helpers. Intended for Phase 3 co-evolution
  stages 23-25 to sample historical opponents.
- sql_tab.py enriched: desc_map covers all monster_species columns,
  training_pairs columns, curriculum_stages monster toggles; _CLUSTERS
  has Monsters cluster; _SOFT_FKS wire monster_species_stats →
  monster_species and monster_species.natural_weapon_key → items
- Meat item subclass: species tag, spoilage tick, cooked/preserved flags
- Monster editor tab (parallel to Species tab)
- Training Pairs table + editor tab: bind creature/goal/monster/pack
  model versions to a named pair with current_stage tracking
- Curriculum stages 15-25 seeded (M_Survive -> M_Eat -> M_Hunt ->
  M_Pack -> M_Dominance -> M_Lifecycle -> C_Predation -> C_Ecosystem
  -> Coevo_A -> Coevo_B -> Final) with freeze toggles
- Monster reward function (monster_reward.py) with 16 signals:
  m_hp, m_hunger, m_kills, m_damage_dealt, m_chase, m_eat, m_graze,
  m_territory_stay, m_territory_hold, m_pack_cohesion, m_sleep_sync,
  m_dominance_wins, m_pair_success, m_reproduction, m_egg_protect,
  m_queen_kill_bonus, m_failed_actions

Monster Runtime
- Monster action dispatch: all 11 actions executed
- Monster tick loop (monster_runtime.monster_tick): batch inference
  via MonsterNet, per-tick action execution, pack event propagation,
  pack housekeeping (splits/merges/challenges)
- Pack NN refresh on slow cadence (2s), broadcasts signals to members
  only when they change beyond epsilon threshold
- Cross-pack combat: hostile packs in contact trade melee attacks
  each tick
- Dominance challenges: contest-type packs roll probabilistic
  challenges; rank swaps on successful challenge
- Pregnancy + egg laying + hatching: gestation timer, monster eggs
  tagged for monster hatch path routes through new Monster spawn
- Arena integration: spawn_monsters_for_stage reads stage subset
  and populates arena['monsters'] + arena['packs']; natural weapons
  auto-equipped on spawn; non-overlapping territory placement
- Simulation.step() drives monster_tick + per-monster process_ticks
  for regen/hunger/water

RL Training Pipeline
- Lifecycle ticks in headless training: _tick_lifecycle_day() fires
  on day boundaries (egg gestation/hatching for creatures + monsters,
  fatigue accumulation, age increment)
- 14-stage creature curriculum + 11-stage monster/coevo curriculum
  with progressive action masking
- Per-stage signal_scales for reward masking with soft fade
- Environment toggles: hunger_drain, combat_enabled, gestation_enabled,
  fatigue_enabled
- Monster environment toggles: monsters_enabled, creature_frozen,
  monster_trainable, pack_trainable, monster_species_subset
- creature_frozen toggle: run_mappo/run_ppo skip the CreatureNet
  and GoalNet gradient step when stage flag is set
- monster_trainable toggle: run_mappo/run_ppo attach a MonsterTrainer
  that loads pretrained .npz (if present), replaces the heuristic
  policy on sim.monster_net, collects rollouts per tick, runs PPO
  updates when the pooled buffer hits rollout_len, and exports
  trained weights back to SAVE_DIR on completion
- Monster injection: _inject_monsters helper adds
  spawn_monsters_for_stage to arenas for stages with monsters_enabled
- 5-layer TorchCreatureNet (1837 -> 1536 -> 1024 -> 768 -> 384 -> 192 -> 32)
- TorchGoalNet for hierarchical purpose selection
- TorchMonsterPolicy + TorchPackPolicy (online RL via MonsterTrainer)
- TorchMonsterNet + TorchPackNet (imitation pretrain)
- PPO with GAE, entropy bonus, gradient clipping
- MAPPO multi-agent + ES evolutionary + PPO single-agent pipeline
- PPO/ES parallel worker support (MAPPO sequential)
- Action mask at logit level (pre-softmax -1e10)
- RolloutBuffer with per-step action masks
- TensorBoard logging: episode reward, entropy, alive_count, per-
  action distribution, per-signal breakdown
- Observation masks applied to NN input (apply_preset_mask in headless
  + train loops)
- Game DB loaded in training arena (dialogue, quests, species data,
  monster species, monster_species_stats)
